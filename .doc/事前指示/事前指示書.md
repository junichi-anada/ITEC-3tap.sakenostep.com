## 3. 設計指示時の注意事項

1. **命名規則**
   - テーブル名：スネークケース（小文字）
   - カラム名：スネークケース（小文字）
   - API エンドポイント：ケバブケース（小文字）

2. **文字コード**
   - UTF-8を使用

3. **コメント**
   - 日本語と英語の併記を推奨
   - 略語は避け、分かりやすい命名を心がける

4. **セキュリティ**
   - パスワードなどの機密情報は必ずハッシュ化
   - 個人情報の取り扱いには十分注意

5. **データベース設計**
   - 論理削除を基本とする（deleted_at カラムを使用）
   - created_at, updated_at は原則全テーブルに付与
   - 外部キー制約は必ず定義

6. **API設計**
   - RESTful APIの原則に従う
   - バージョニングを必ず含める
   - エラーレスポンスは統一フォーマットで返す

## 5. 設計変更・追加時の指示方法

1. **テーブル追加**
```markdown
@テーブル追加
機能名: [機能名]
テーブル名: [テーブル名]
説明: [説明]
```

2. **カラム追加**
```markdown
@カラム追加
テーブル名: [テーブル名]
カラム名: [カラム名]
型: [型]
制約: [制約]
説明: [説明]
```

3. **API追加**
```markdown
@API追加
機能名: [機能名]
エンドポイント: [エンドポイント]
メソッド: [HTTPメソッド]
説明: [説明]
```

これらの指示に従って、一貫性のある設計支援を行ってください。
