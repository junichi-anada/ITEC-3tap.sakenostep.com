# お気に入り商品一覧機能設計書

## **1. 機能概要**
- **目的**: ユーザーが自身で登録したお気に入り商品の一覧を表示する機能。

## **2. 要件一覧**
### **必須要件**
- ユーザーごとに登録されたお気に入り商品を取得可能。
- 検索結果は価格順に並び替え可能（昇順/降順）。
- 無限スクロール対応（ページングではなく、必要なデータを逐次取得）。

### **拡張要件**
- カテゴリで絞り込み可能。

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                                   |
|------------------|----------|------|----------------------------------------|
| `user_id`        | 整数     | 必須 | 対象のユーザーID                      |
| `category_id`    | 整数     | 任意 | 絞り込み対象のカテゴリID              |
| `sort`           | 文字列   | 任意 | 並び替え条件（例: `price_asc`, `price_desc`）|
| `offset`         | 整数     | 任意 | データの取得開始位置                  |
| `limit`          | 整数     | 任意 | 取得するデータ数（デフォルト: 10）     |

### **出力**
#### **正常系**
- **HTTPステータス**: 200 OK
- **レスポンス例**:
```json
{
    "status": "success",
    "data": {
        "items": [
            {
                "id": 1,
                "item_code": "A001",
                "name": "商品A",
                "price": 1000,
                "category": "カテゴリ1",
                "published_at": "2025-01-01"
            },
            {
                "id": 2,
                "item_code": "A002",
                "name": "商品B",
                "price": 1500,
                "category": "カテゴリ2",
                "published_at": "2025-01-02"
            }
        ],
        "next_offset": 20
    }
}
```

#### **異常系**
- **パラメータが無効な場合**:
  - HTTPステータス: 400 Bad Request
  - レスポンス例:
```json
{
    "status": "error",
    "message": "入力パラメータが不正です。"
}
```

## **4. ロジック/処理フロー**
1. ユーザーが `user_id`（必須）、`category_id`（任意）、`sort`（任意）、`offset`、`limit` をリクエストとして送信。
2. `favorite_items` テーブルと `items` テーブルを結合し、対象ユーザーのお気に入り商品を取得。
    - 指定があればカテゴリで絞り込み。
    - 取得結果を並び替え（価格昇順/降順）。
    - 指定範囲の商品を抽出し、`next_offset` を計算。
3. 検索結果をレスポンスとして返却。

## **5. 関連テーブル/データ構造**
- **`favorite_items` テーブル**
  | カラム名         | 型       | 説明                       |
  |------------------|----------|----------------------------|
  | `id`             | INT      | ユニークID                |
  | `user_id`        | INT      | ユーザーID                |
  | `item_id`        | INT      | 商品ID                    |
  | `created_at`     | DATETIME | 作成日時                  |

- **`items` テーブル**
  | カラム名         | 型       | 説明                       |
  |------------------|----------|----------------------------|
  | `id`             | INT      | ユニークID                |
  | `item_code`      | VARCHAR  | 商品コード（ユニーク）     |
  | `name`           | VARCHAR  | 商品名                    |
  | `price`          | DECIMAL  | 商品価格                  |
  | `category_id`    | INT      | 商品カテゴリID            |
  | `published_at`   | DATETIME | 公開日時                  |
  | `is_recommended` | BOOLEAN  | おすすめ商品フラグ        |
  | `created_at`     | DATETIME | 作成日時                  |
  | `updated_at`     | DATETIME | 更新日時                  |

## **6. バリデーション仕様**
- `user_id`: 必須、正の整数。
- `category_id`: 正の整数。
- `sort`: `price_asc`, `price_desc` のいずれか。
- `offset`: 0以上の整数。
- `limit`: 正の整数。

## **7. エラーハンドリング**
- 無効なパラメータの場合、400エラーを返却。
- 検索結果が0件の場合、空のリストを返却（エラーにはしない）。

## **8. API仕様**
### **エンドポイント**
- **URL**: `/api/users/{user_id}/favorites`
- **HTTPメソッド**: GET

### **リクエスト例**
```http
GET /api/users/1/favorites?category_id=1&sort=price_asc&offset=0&limit=10
```

### **レスポンス例**
#### 正常系
```json
{
    "status": "success",
    "data": {
        "items": [
            {
                "id": 1,
                "item_code": "A001",
                "name": "商品A",
                "price": 1000,
                "category": "カテゴリ1",
                "published_at": "2025-01-01"
            }
        ],
        "next_offset": 10
    }
}
```

#### 異常系
```json
{
    "status": "error",
    "message": "入力パラメータが不正です。"
}
