# お気に入り商品削除機能設計書

## **1. 機能概要**
- **目的**: ユーザーが登録済みのお気に入り商品を削除できる機能。

## **2. 要件一覧**
### **必須要件**
- ユーザーが登録済みのお気に入り商品を削除できる。
- 削除対象が存在しない場合、エラーメッセージを返却する。
- 削除成功時、「削除完了」のメッセージを返却する。

### **拡張要件**
- 複数の商品を一括で削除できる。
- 削除履歴を記録する。

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                      |
|------------------|----------|------|---------------------------|
| `user_id`        | 整数     | 必須 | ユーザーID               |
| `item_id`        | 整数     | 必須 | 削除する商品ID            |

### **出力**
#### **正常系**
- **HTTPステータス**: 200 OK
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "お気に入りから削除しました。"
}
```

#### **異常系**
- **削除対象が存在しない場合**:
  - HTTPステータス: 404 Not Found
  - レスポンス例:
```json
{
    "status": "error",
    "message": "削除対象の商品が見つかりません。"
}
```

## **4. ロジック/処理フロー**
1. ユーザーが `user_id` と `item_id` をリクエストとして送信する。
2. `favorite_items` テーブルを検索し、削除対象が存在するか確認。
    - 存在しない場合、エラーレスポンスを返却。
3. `favorite_items` テーブルから該当レコードを削除。
4. 削除成功レスポンスを返却。

## **5. 関連テーブル/データ構造**
- **`favorite_items` テーブル**
  | カラム名       | 型       | 説明                       |
  |----------------|----------|----------------------------|
  | `id`           | INT      | ユニークID                |
  | `user_id`      | INT      | ユーザーID                |
  | `item_id`      | INT      | 商品ID                    |
  | `site_id`      | INT      | サイトID                  |
  | `created_at`   | DATETIME | 作成日時                  |
  | `deleted_at`   | DATETIME | 削除日時（論理削除用）     |

## **6. バリデーション仕様**
- `user_id`: 必須、正の整数。
- `item_id`: 必須、正の整数。

## **7. エラーハンドリング**
- 存在しない商品IDが指定された場合、404エラーを返却。
- ユーザーに紐づかない商品IDが指定された場合、エラーメッセージを返却。

## **8. API仕様**
### **エンドポイント**
- **URL**: `/api/favorites`
- **HTTPメソッド**: DELETE

### **リクエスト例**
```json
{
    "user_id": 1,
    "item_id": 42
}
```

### **レスポンス例**
#### 正常系
```json
{
    "status": "success",
    "message": "お気に入りから削除しました。"
}
```

#### 異常系
```json
{
    "status": "error",
    "message": "削除対象の商品が見つかりません。"
}
