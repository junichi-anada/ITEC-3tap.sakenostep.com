# 商品検索機能設計書

## **1. 機能概要**
- **目的**: ユーザーが商品名やカテゴリを基に商品を検索できる機能。

## **2. 要件一覧**
### **必須要件**
- 商品名の部分一致で検索可能。
- 商品カテゴリで絞り込み可能。
- 検索結果は価格順に並び替え可能（昇順/降順）。
- 無限スクロール対応（ページングではなく、必要なデータを逐次取得）。
- 商品コードを指定して個別の商品詳細を取得可能。
- おすすめ商品のみを取得可能。

### **拡張要件**
- 商品の在庫状況で絞り込み可能。
- ユーザーの検索履歴を保存。

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                                   |
|------------------|----------|------|----------------------------------------|
| `query`          | 文字列   | 任意 | 検索する商品名（部分一致）            |
| `item_code`      | 文字列   | 任意 | 商品コード（個別商品の取得に使用）     |
| `category_id`    | 整数     | 任意 | 絞り込み対象のカテゴリID              |
| `is_recommended` | ブール型 | 任意 | おすすめ商品のみを取得する場合に指定  |
| `sort`           | 文字列   | 任意 | 並び替え条件（例: `price_asc`, `price_desc`）|
| `offset`         | 整数     | 任意 | データの取得開始位置                  |
| `limit`          | 整数     | 任意 | 取得するデータ数（デフォルト: 10）     |

### **出力**
#### **正常系**
- **商品リスト取得**:
  - **HTTPステータス**: 200 OK
  - **レスポンス例**:
```json
{
    "status": "success",
    "data": {
        "items": [
            {
                "id": 1,
                "item_code": "A001",
                "name": "商品A",
                "price": 1000,
                "category": "カテゴリ1",
                "published_at": "2025-01-01",
                "is_recommended": true
            },
            {
                "id": 2,
                "item_code": "A002",
                "name": "商品B",
                "price": 1500,
                "category": "カテゴリ2",
                "published_at": "2025-01-02",
                "is_recommended": false
            }
        ],
        "next_offset": 20
    }
}
```
- **個別商品取得**:
  - **HTTPステータス**: 200 OK
  - **レスポンス例**:
```json
{
    "status": "success",
    "data": {
        "id": 1,
        "item_code": "A001",
        "name": "商品A",
        "price": 1000,
        "category_id": 1,
        "published_at": "2025-01-01",
        "is_recommended": true
    }
}
```

#### **異常系**
- **パラメータが無効な場合**:
  - HTTPステータス: 400 Bad Request
  - レスポンス例:
```json
{
    "status": "error",
    "message": "入力パラメータが不正です。"
}
```
- **対象の商品が存在しない場合**:
  - HTTPステータス: 404 Not Found
  - レスポンス例:
```json
{
    "status": "error",
    "message": "対象の商品が見つかりません。"
}
```

## **4. ロジック/処理フロー**
1. ユーザーが検索条件（`query`, `category_id`, `is_recommended`, `sort`, `offset`, `limit`）または商品コード（`item_code`）をリクエストとして送信。
2. リクエスト内容に応じて以下を実行:
    - **商品リスト取得**:
        1. `items` テーブルを検索し、条件に一致する商品を取得。
        2. 取得結果をソート（価格昇順/降順）。
        3. 指定範囲の商品を抽出し、`next_offset` を計算。
    - **個別商品取得**:
        1. 商品コードに一致する商品を検索。
        2. 存在しない場合、404エラーを返却。
3. 結果をレスポンスとして返却。

## **5. 関連テーブル/データ構造**
- **`items` テーブル**
  | カラム名         | 型       | 説明                       |
  |------------------|----------|----------------------------|
  | `id`             | INT      | ユニークID                |
  | `item_code`      | VARCHAR  | 商品コード（ユニーク）     |
  | `name`           | VARCHAR  | 商品名                    |
  | `price`          | DECIMAL  | 商品価格                  |
  | `category_id`    | INT      | 商品カテゴリID            |
  | `published_at`   | DATETIME | 公開日時                  |
  | `is_recommended` | BOOLEAN  | おすすめ商品フラグ        |
  | `created_at`     | DATETIME | 作成日時                  |
  | `updated_at`     | DATETIME | 更新日時                  |

## **6. バリデーション仕様**
- `query`: 最大50文字。
- `item_code`: 最大20文字、ユニーク。
- `category_id`: 正の整数。
- `is_recommended`: 真偽値。
- `sort`: `price_asc`, `price_desc` のいずれか。
- `offset`: 0以上の整数。
- `limit`: 正の整数。

## **7. エラーハンドリング**
- 無効なパラメータの場合、400エラーを返却。
- 検索結果が0件の場合、空のリストを返却（エラーにはしない）。
- 商品コードが存在しない場合、404エラーを返却。

## **8. API仕様**
### **エンドポイント**
- **商品リスト取得**:
  - **URL**: `/api/items`
  - **HTTPメソッド**: GET
- **個別商品取得**:
  - **URL**: `/api/items/{item_code}`
  - **HTTPメソッド**: GET

### **リクエスト例**
#### 商品リスト取得
```http
GET /api/items?query=商品&category_id=1&is_recommended=true&sort=price_asc&offset=0&limit=10
```
#### 個別商品取得
```http
GET /api/items/A001
```
