# 商品登録機能設計書

## **1. 機能概要**
- **目的**: オペレータが新しい商品を手動で登録できる機能。

## **2. 要件一覧**
### **必須要件**
- 商品コード、商品名、価格、カテゴリを登録できる。
- 商品コードの重複登録は防ぐ。
- 商品が公開されるかどうかを指定できる。
- 商品の登録方法（手動/インポート）を記録。

### **拡張要件**
- 登録時に商品画像をアップロードできる。
- 複数のカテゴリに所属できる。

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                                   |
|------------------|----------|------|----------------------------------------|
| `item_code`      | 文字列   | 必須 | 商品コード（ユニーク）                |
| `name`           | 文字列   | 必須 | 商品名                                 |
| `price`          | 小数     | 必須 | 商品価格（0以上の値）                  |
| `category_id`    | 整数     | 必須 | 商品カテゴリID                         |
| `published_at`   | 日付     | 任意 | 公開開始日時（未指定の場合は非公開）   |
| `source`         | 文字列   | 必須 | 登録方法（例: MANUAL）                 |
| `is_recommended` | ブール型 | 任意 | おすすめ商品かどうか                   |

### **出力**
#### **正常系**
- **HTTPステータス**: 201 Created
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "商品を登録しました。",
    "data": {
        "id": 1,
        "item_code": "A001",
        "name": "商品A",
        "price": 1000,
        "category_id": 1,
        "published_at": "2025-01-01",
        "source": "MANUAL",
        "is_recommended": false
    }
}
```

#### **異常系**
- **商品コードが重複している場合**:
  - HTTPステータス: 400 Bad Request
  - レスポンス例:
```json
{
    "status": "error",
    "message": "商品コードが既に存在します。"
}
```
- **入力パラメータが無効な場合**:
  - HTTPステータス: 400 Bad Request
  - レスポンス例:
```json
{
    "status": "error",
    "message": "入力パラメータが不正です。"
}
```

## **4. ロジック/処理フロー**
1. オペレータが商品情報を入力し、登録リクエストを送信。
2. 入力データのバリデーションを実施。
    - 必須項目が全て入力されているか確認。
    - 商品コードが既に存在する場合はエラーを返却。
3. バリデーションに成功した場合、`items` テーブルに商品情報を登録。
4. 登録成功時に、登録内容を含むレスポンスを返却。

## **5. 関連テーブル/データ構造**
- **`items` テーブル**
  | カラム名         | 型       | 説明                       |
  |------------------|----------|----------------------------|
  | `id`             | INT      | ユニークID                |
  | `item_code`      | VARCHAR  | 商品コード（ユニーク）     |
  | `name`           | VARCHAR  | 商品名                    |
  | `price`          | DECIMAL  | 商品価格                  |
  | `category_id`    | INT      | 商品カテゴリID            |
  | `published_at`   | DATETIME | 公開日時                  |
  | `source`         | ENUM     | 登録方法（例: MANUAL）     |
  | `is_recommended` | BOOLEAN  | おすすめ商品フラグ        |
  | `created_at`     | DATETIME | 作成日時                  |
  | `updated_at`     | DATETIME | 更新日時                  |

## **6. バリデーション仕様**
- `item_code`: 最大20文字、ユニーク。
- `name`: 最大100文字。
- `price`: 0以上の値。
- `category_id`: 正の整数。
- `published_at`: 有効な日付形式。
- `source`: `MANUAL` または `IMPORT` のいずれか。
- `is_recommended`: 真偽値。

## **7. エラーハンドリング**
- 商品コードが重複している場合、400エラーを返却。
- 無効な入力パラメータの場合、400エラーを返却。

## **8. API仕様**
### **エンドポイント**
- **URL**: `/api/items`
- **HTTPメソッド**: POST

### **リクエスト例**
```json
{
    "item_code": "A001",
    "name": "商品A",
    "price": 1000,
    "category_id": 1,
    "published_at": "2025-01-01",
    "source": "MANUAL",
    "is_recommended": false
}
```

### **レスポンス例**

    "data": {
        "id": 1,
