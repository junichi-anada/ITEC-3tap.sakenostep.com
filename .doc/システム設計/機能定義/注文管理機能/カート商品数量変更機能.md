# カート商品数量変更機能設計書

## **1. 機能概要**
- **目的**: カートに追加済みの商品の数量を変更する機能
- **対象者**: システムにログインした顧客
- **方式**: データベースとローカルストレージを併用したハイブリッド方式

## **2. 要件一覧**
### **必須要件**
- 単一商品の数量を変更できる
- 複数商品の数量を一括で変更できる
- カートデータをデータベースで管理
- カートデータをローカルストレージにキャッシュ

### **拡張要件**
- 在庫状況のリアルタイム確認
- 数量変更時の小計・合計金額の自動計算
- オフライン時の一時的な数量変更対応

## **3. 入出力仕様**
### **入力**
#### 単一商品数量更新
```json
{
    "item_id": 1,
    "quantity": 3
}
```

#### 複数商品一括数量更新
```json
{
    "items": [
        {
            "item_id": 1,
            "quantity": 3
        },
        {
            "item_id": 2,
            "quantity": 2
        }
    ]
}
```

| 項目           | 型          | 必須 | 説明                          |
|---------------|-------------|------|------------------------------|
| `item_id`     | 整数        | ※1  | 商品ID（単一更新時）          |
| `quantity`    | 整数        | ※1  | 数量（1以上、単一更新時）      |
| `items`       | 配列        | ※1  | 商品リスト（一括更新時）       |
| `items[].item_id` | 整数    | ※2  | 商品ID                       |
| `items[].quantity` | 整数   | ※2  | 数量（1以上）                |

※1: `item_id`と`quantity`、または`items`のいずれかが必須
※2: `items`配列使用時は必須

### **出力**
#### 単一商品数量更新時
```json
{
    "status": "success",
    "message": "数量を更新しました",
    "data": {
        "cart_id": 1,
        "item_id": 1,
        "name": "商品A",
        "price": 1000,
        "quantity": 3,
        "total": 3000,
        "cache_timestamp": "2024-03-01T10:00:00Z"
    }
}
```

#### 複数商品一括数量更新時
```json
{
    "status": "success",
    "message": "数量を更新しました",
    "data": {
        "cart_id": 1,
        "items": [
            {
                "item_id": 1,
                "name": "商品A",
                "price": 1000,
                "quantity": 3,
                "total": 3000
            },
            {
                "item_id": 2,
                "name": "商品B",
                "price": 1500,
                "quantity": 2,
                "total": 3000
            }
        ],
        "total_amount": 6000,
        "cache_timestamp": "2024-03-01T10:00:00Z"
    }
}
```

## **4. 処理フロー**
1. 数量更新処理
   - リクエストパラメータの形式チェック
   - 単一更新か一括更新かの判定
   - 各商品に対して:
     - 対象商品の存在チェック
     - 数量のバリデーション
   - トランザクション開始
   - 注文明細の一括更新
   - 合計金額の再計算
   - トランザクション確定
   - キャッシュの更新

## **5. API仕様**
### エンドポイント
- **カート内商品数量更新**:
  - 単一商品更新:
    - URL: `/api/cart/items/{item_id}`
    - メソッド: PUT
  - 複数商品一括更新:
    - URL: `/api/cart/items`
    - メソッド: PUT
  - Content-Type: application/json

### リクエストヘッダー
```
Authorization: Bearer {token}
X-CSRF-TOKEN: {csrf_token}
```

## **6. データ設計**
### データベース設計
| カラム名      | 型      | 説明                    |
|--------------|---------|------------------------|
| id           | INT     | 注文ID                 |
| user_id      | INT     | ユーザーID             |
| status       | ENUM    | 注文状態（CART/ORDERED）|
| total_amount | DECIMAL | 合計金額               |
| created_at   | DATETIME| 作成日時               |
| updated_at   | DATETIME| 更新日時               |

### ローカルストレージ設計
```json
{
    "cart": {
        "id": 1,
        "items": [
            {
                "item_id": 1,
                "name": "商品A",
                "price": 1000,
                "quantity": 3
            }
        ],
        "total_amount": 3000,
        "timestamp": "2024-03-01T10:00:00Z",
        "expires_at": "2024-03-01T11:00:00Z"
    }
}
```

## **7. バリデーション仕様**
- `item_id`: 必須、カートに存在する商品ID
- `quantity`: 必須、1以上の整数
- 在庫数チェック
- 最大注文数チェック

## **8. エラーハンドリング**
### バリデーションエラー
```json
{
    "status": "error",
    "message": "数量の更新に失敗しました",
    "errors": {
        "items.0.quantity": ["1以上の数値を入力してください"],
        "items.1.item_id": ["カートに存在しない商品です"]
    }
}
```

### 在庫不足エラー
```json
{
    "status": "error",
    "message": "在庫が不足しています",
    "errors": {
        "items.0.quantity": ["在庫数が不足しています（在庫数: 5）"]
    }
}
```

## **9. セキュリティ対策**
- 認証済みユーザーのみアクセス可能
- CSRFトークンによる保護
- 入力値のサニタイズ
- ユーザーごとのカート情報の分離
- キャッシュデータの暗号化
- キャッシュの有効期限設定
