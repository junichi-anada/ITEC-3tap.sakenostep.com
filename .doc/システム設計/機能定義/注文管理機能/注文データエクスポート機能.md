# 注文データエクスポート機能設計書

## **1. 機能概要**
- **目的**: オペレータが注文データをCSVファイルとして出力する機能
- **対象者**: システムのオペレータ
- **方式**: CSVファイル出力方式（Shift-JIS形式）

## **2. 要件一覧**
### **必須要件**
- 注文データをCSVファイルとして出力可能
- 出力済みの注文データは出力済みフラグを設定
- CSVファイルは自動でダウンロード開始
- 検索条件に基づいた注文データの絞り込み出力

### **拡張要件**
- エクスポート履歴の表示
- エラー発生時のログ記録
- 大量データ時の非同期処理対応

## **3. 入出力仕様**
### **入力**
- CSRFトークン（セキュリティ対策）
- その他の入力パラメータは不要（未出力データを全て出力）

### **出力**
#### **CSVファイル形式**
| 項目 | 説明 |
|------|------|
| 取引先区分 | 1固定 |
| 伝票日付 | 注文日（YYYYMMDD形式） |
| 取引先コード | 顧客コード（5桁） |
| 納品先コード | 1固定 |
| 伝票行番区分 | 1固定 |
| 商品コード | 商品の管理コード |
| JANコード | 商品のJANコード |
| ケース／バラ | ケース単位の場合1、バラの場合0 |
| 数量 | 注文数量 |
| 税区分 | （空欄） |
| 単価 | （空欄） |
| 金額 | （空欄） |
| 相手先伝票番号 | 注文コード |
| 相手先商品コード | 商品コード |
| 相手先商品名 | 商品名（15文字まで） |

## **4. 処理フロー**
1. エクスポート開始
   - CSRFトークンの検証のみ

2. データ取得処理
   - exported_atがnullの注文データを全件取得
   - 出力対象データの整形

3. CSV出力処理
   - BOM付きShift-JISでのエンコード
   - ヘッダー行の出力
   - データ行の出力

4. 後処理
   - 出力済みフラグの更新
   - エクスポートログの記録
   - ファイルのダウンロード開始

## **5. データ構造**
### **import_export_logs テーブル**
| カラム名 | 型 | 説明 |
|----------|-----|------|
| `id` | INT | ログID |
| `log_code` | VARCHAR(64) | ログコード |
| `site_id` | INT | サイトID |
| `operator_id` | INT | 操作者ID |
| `operation_type` | ENUM | 操作種別（EXPORT固定） |
| `target_type` | ENUM | 対象種別（ORDER固定） |
| `file_name` | VARCHAR(255) | 出力ファイル名 |
| `record_count` | INT | 出力レコード数 |
| `status` | ENUM | 処理状態 |
| `error_detail` | TEXT | エラー詳細 |
| `created_at` | DATETIME | 作成日時 |

### **orders テーブル**
| カラム名 | 型 | 説明 |
|----------|-----|------|
| `exported_at` | DATETIME | CSV出力日時（nullの場合は未出力） |

## **6. バリデーション仕様**
- CSRFトークンの存在確認

## **7. エラーハンドリング**
### バリデーションエラー
- HTTPステータス: 422 Unprocessable Entity
- エラーメッセージを返却

### システムエラー
- HTTPステータス: 500 Internal Server Error
- エラーログに記録
- エラーメッセージを返却

## **8. セキュリティ対策**
- CSRFトークンによる保護
- 実行権限の確認
- 出力ファイルの一時保存と適切な削除
- データ件数の制限（1回の出力上限設定）

## **9. パフォーマンス対策**
- データベースクエリの最適化
- 大量データ時のメモリ使用量制御
- 一時ファイルの適切な管理
- クエリ実行時のインデックス活用

## **10. テストケース**
### 正常系
1. 未出力データの一括出力
2. 出力対象データが0件の場合の処理
3. 大量の未出力データがある場合の処理

### 異常系
1. CSRFトークンが無効な場合
2. 認証が切れている場合
3. 権限がない場合
4. 大量データ出力時の動作確認
