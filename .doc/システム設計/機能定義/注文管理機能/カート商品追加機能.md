# カート商品追加機能設計書

## **1. 機能概要**
- **目的**: 顧客が選択した商品を注文カートに追加する機能
- **対象者**: システムにログインした顧客
- **方式**: データベースとローカルストレージを併用したハイブリッド方式

## **2. 要件一覧**
### **必須要件**
- 単一商品をカートに追加できる
- 複数商品を一括でカートに追加できる
- カートデータをデータベースで管理
- カートデータをローカルストレージにキャッシュ

### **拡張要件**
- 在庫状況のリアルタイム確認
- 追加時の小計・合計金額の自動計算
- オフライン時の一時的な追加操作対応

## **3. 入出力仕様**
### **入力**
#### 単一商品追加
```json
{
    "item_id": 1,
    "quantity": 2
}
```

#### 複数商品一括追加
```json
{
    "items": [
        {
            "item_id": 1,
            "quantity": 2
        },
        {
            "item_id": 2,
            "quantity": 1
        }
    ]
}
```

| 項目           | 型          | 必須 | 説明                          |
|---------------|-------------|------|------------------------------|
| `item_id`     | 整数        | ※1  | 商品ID（単一追加時）          |
| `quantity`    | 整数        | ※1  | 数量（1以上、単一追加時）      |
| `items`       | 配列        | ※1  | 商品リスト（一括追加時）       |
| `items[].item_id` | 整数    | ※2  | 商品ID                       |
| `items[].quantity` | 整数   | ※2  | 数量（1以上）                |

※1: `item_id`と`quantity`、または`items`のいずれかが必須
※2: `items`配列使用時は必須

### **出力**
#### 単一商品追加時
```json
{
    "status": "success",
    "message": "カートに追加しました",
    "data": {
        "cart_id": 1,
        "item_id": 1,
        "name": "商品A",
        "price": 1000,
        "quantity": 2,
        "total": 2000,
        "cache_timestamp": "2024-03-01T10:00:00Z"
    }
}
```

#### 複数商品一括追加時
```json
{
    "status": "success",
    "message": "カートに追加しました",
    "data": {
        "cart_id": 1,
        "items": [
            {
                "item_id": 1,
                "name": "商品A",
                "price": 1000,
                "quantity": 2,
                "total": 2000
            },
            {
                "item_id": 2,
                "name": "商品B",
                "price": 1500,
                "quantity": 1,
                "total": 1500
            }
        ],
        "total_amount": 3500,
        "cache_timestamp": "2024-03-01T10:00:00Z"
    }
}
```

## **4. 処理フロー**
1. 商品追加処理
   - 商品の存在チェック
   - 数量のバリデーション
   - 未確定の注文（カート）の取得または新規作成
   - 注文明細（OrderDetails）の追加・更新
   - 合計金額の計算と更新
   - ローカルストレージへのキャッシュ保存

## **5. API仕様**
### エンドポイント
- **カートへの商品追加**:
  - URL: `/api/cart/items`
  - メソッド: POST
  - Content-Type: application/json

### リクエストヘッダー
```
Authorization: Bearer {token}
X-CSRF-TOKEN: {csrf_token}
```

## **6. データ設計**
### データベース設計
| カラム名      | 型      | 説明                    |
|--------------|---------|------------------------|
| id           | INT     | 注文ID                 |
| user_id      | INT     | ユーザーID             |
| status       | ENUM    | 注文状態（CART/ORDERED）|
| total_amount | DECIMAL | 合計金額               |
| created_at   | DATETIME| 作成日時               |
| updated_at   | DATETIME| 更新日時               |

### ローカルストレージ設計
```json
{
    "cart": {
        "id": 1,
        "items": [
            {
                "item_id": 1,
                "name": "商品A",
                "price": 1000,
                "quantity": 2
            }
        ],
        "total_amount": 2000,
        "timestamp": "2024-03-01T10:00:00Z",
        "expires_at": "2024-03-01T11:00:00Z"
    }
}
```

## **7. バリデーション仕様**
- `item_id`: 必須、存在する商品ID
- `quantity`: 必須、1以上の整数
- 在庫数チェック
- 最大注文数チェック

## **8. エラーハンドリング**
### バリデーションエラー
```json
{
    "status": "error",
    "message": "商品の追加に失敗しました",
    "errors": {
        "items.0.quantity": ["1以上の数値を入力してください"],
        "items.1.item_id": ["存在しない商品IDです"]
    }
}
```

### 在庫不足エラー
```json
{
    "status": "error",
    "message": "在庫が不足しています",
    "errors": {
        "items.0.quantity": ["在庫数が不足しています（在庫数: 5）"]
    }
}
```

## **9. セキュリティ対策**
- 認証済みユーザーのみアクセス可能
- CSRFトークンによる保護
- 入力値のサニタイズ
- ユーザーごとのカート情報の分離
- キャッシュデータの暗号化
- キャッシュの有効期限設定
