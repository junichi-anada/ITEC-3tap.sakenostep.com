# お気に入り商品登録機能設計書

## **1. 機能概要**
- **目的**: ユーザーが商品の詳細ページやリストページから商品を「お気に入り」に登録できる機能。

## **2. 要件一覧**
### **必須要件**
- ユーザーが商品をお気に入りに登録できる。
- 同じ商品を重複してお気に入り登録することはできない。
- 登録に成功した場合、「登録完了」のメッセージを返却する。

### **拡張要件**
- お気に入り登録の上限数を設定できる。
- 商品リストにお気に入り登録済みのバッジを表示する。

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                      |
|------------------|----------|------|---------------------------|
| `user_id`        | 整数     | 必須 | ユーザーID               |
| `item_id`        | 整数     | 必須 | 登録する商品ID            |

### **出力**
#### **正常系**
- **HTTPステータス**: 200 OK
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "お気に入りに登録しました。"
}
```

#### **異常系**
- **重複登録時**:
  - HTTPステータス: 400 Bad Request
  - レスポンス例:
```json
{
    "status": "error",
    "message": "この商品はすでにお気に入りに登録されています。"
}
```
- **登録上限時**:
  - HTTPステータス: 400 Bad Request
  - レスポンス例:
```json
{
    "status": "error",
    "message": "お気に入り登録の上限に達しています。"
}
```

## **4. ロジック/処理フロー**
1. ユーザーが `user_id` と `item_id` をリクエストとして送信する。
2. `favorite_items` テーブルを検索し、以下を確認:
    - 有効なレコードが存在する場合（deleted_at が NULL）、エラーレスポンスを返却。
    - 論理削除されたレコードが存在する場合（deleted_at が NOT NULL）、そのレコードを復活させる。
    - レコードが存在しない場合、新規登録を行う。
3. 登録数の上限を確認（設定がある場合）。
    - 上限を超える場合、エラーレスポンスを返却。
4. レコードの状態に応じて以下の処理を実行:
    - 新規登録の場合: `favorite_items` テーブルに新しいレコードを挿入。
    - 論理削除済みの場合: deleted_at を NULL に更新し、updated_at を現在時刻に更新。
5. 登録成功レスポンスを返却。

## **5. 関連テーブル/データ構造**
- **`favorite_items` テーブル**
  | カラム名       | 型       | 説明                       |
  |----------------|----------|----------------------------|
  | `id`           | INT      | ユニークID                |
  | `user_id`      | INT      | ユーザーID                |
  | `item_id`      | INT      | 商品ID                    |
  | `site_id`      | INT      | サイトID                  |
  | `created_at`   | DATETIME | 作成日時                  |
  | `updated_at`   | DATETIME | 更新日時                  |
  | `deleted_at`   | DATETIME | 削除日時（論理削除用）     |

## **6. バリデーション仕様**
- `user_id`: 必須、正の整数。
- `item_id`: 必須、正の整数。

## **7. エラーハンドリング**
- 有効な状態で登録済みの商品の重複登録は防ぐ。
- 論理削除された商品の場合は、登録を許可し既存レコードを復活させる。
- 上限を超える登録は防ぐ。
- 存在しない商品IDやユーザーIDが指定された場合、エラーレスポンスを返却。

## **8. API仕様**
### **エンドポイント**
- **URL**: `/api/favorites`
- **HTTPメソッド**: POST

### **リクエスト例**
```json
{
    "user_id": 1,
    "item_id": 42
}
```

### **レスポンス例**
#### 正常系
```json
{
    "status": "success",
    "message": "お気に入りに登録しました。"
}
```

#### 異常系
```json
{
    "status": "error",
    "message": "この商品はすでにお気に入りに登録されています。"
}
