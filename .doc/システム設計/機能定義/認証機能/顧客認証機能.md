# 顧客認証機能設計書

## **1. 機能概要**
- **目的**: 顧客が「お客様コード」と「電話番号」で認証を行い、システムにログインできるようにする機能。

## **2. 要件一覧**
### **必須要件**
- お客様コードと電話番号を用いた認証を実施。
- 認証成功時に認証トークン（例: JWT）を発行。
- 認証失敗時にエラーメッセージを返却。
- トークンに有効期限を設定。

### **拡張要件**
- 2要素認証（2FA）への対応。
- ログイン試行失敗時のアカウントロック機能。

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                       |
|------------------|----------|------|----------------------------|
| `customer_code`  | 文字列   | 必須 | 顧客のお客様コード         |
| `phone_number`   | 文字列   | 必須 | 顧客の電話番号             |

### **出力**
#### **正常系**
- **HTTPステータス**: 200 OK
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "ログイン成功",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "token_type": "Bearer",
        "expires_in": 3600
    }
}
```

#### **異常系**
- **認証失敗時**:
  - HTTPステータス: 401 Unauthorized
  - レスポンス例:
```json
{
    "status": "error",
    "message": "認証に失敗しました。お客様コードまたは電話番号が正しくありません。"
}
```
- **アカウントロック時**:
  - HTTPステータス: 403 Forbidden
  - レスポンス例:
```json
{
    "status": "error",
    "message": "アカウントがロックされています。管理者にお問い合わせください。"
}
```

## **4. ロジック/処理フロー**
1. 顧客が「お客様コード」と「電話番号」をリクエストとして送信。
2. `authenticates` テーブルから該当するレコードを検索。
    - お客様コードが存在しない場合、401エラーを返却。
3. 電話番号を照合。
    - 一致しない場合、401エラーを返却。
4. アカウントがロックされている場合、403エラーを返却。
5. 認証成功時、以下を実行:
    - 認証トークンを生成。
    - トークンの有効期限を設定。
    - トークンをレスポンスとして返却。

## **5. 関連テーブル/データ構造**
- **`authenticates` テーブル**
  | カラム名         | 型       | 説明                       |
  |------------------|----------|----------------------------|
  | `id`             | INT      | 認証レコードID            |
  | `customer_code`  | VARCHAR  | 顧客のお客様コード         |
  | `phone_number`   | VARCHAR  | 顧客の電話番号             |
  | `is_locked`      | BOOLEAN  | 認証ロックフラグ           |
  | `created_at`     | DATETIME | 作成日時                  |
  | `updated_at`     | DATETIME | 更新日時                  |

## **6. バリデーション仕様**
- `customer_code`: 必須、最大20文字。
- `phone_number`: 必須、電話番号形式。

## **7. エラーハンドリング**
- お客様コードが存在しない場合、401エラーを返却。
- 電話番号が一致しない場合、401エラーを返却。
- アカウントがロックされている場合、403エラーを返却。
- 無効な入力パラメータの場合、400エラーを返却。

## **8. API仕様**
### **エンドポイント**
- **URL**: `/api/auth/login/customer`
- **HTTPメソッド**: POST

### **リクエスト例**
```json
{
    "customer_code": "CUST12345",
    "phone_number": "09012345678"
}
```

### **レスポンス例**
#### 正常系
```json
{
    "status": "success",
    "message": "ログイン成功",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "token_type": "Bearer",
        "expires_in": 3600
    }
}
```

#### 異常系
```json
{
    "status": "error",
    "message": "認証に失敗しました。お客様コードまたは電話番号が正しくありません。"
