# 顧客データインポート機能設計書

## **1. 機能概要**
- **目的**: オペレータが顧客データを一括でインポートできる機能。
- **対象者**: システムのオペレータ
- **方式**: ファイルアップロード方式（Excel/CSV/TXT対応）

## **2. 要件一覧**
### **必須要件**
- Excel（xls/xlsx）、テキストファイル（csv/txt）形式のファイルをインポート可能
- ファイルサイズは最大10MBまで
- インポート結果をログとして記録
- インポート処理はバックグラウンドで実行
- 処理結果をオペレータに通知
- 重複データのチェックと処理（電話番号による重複チェック）

### **拡張要件**
- インポート履歴の表示
- エラーデータの出力機能
- インポートテンプレートのダウンロード機能
- 処理進捗状況のリアルタイム表示

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                                |
|------------------|----------|------|-------------------------------------|
| `file`           | File     | 必須 | インポートファイル（10MB以内）      |
| `_token`         | 文字列   | 必須 | CSRFトークン                        |

### **CSVファイル形式**
| カラム名         | 型       | 必須 | 説明                       |
|------------------|----------|------|----------------------------|
| `name`           | 文字列   | 必須 | 顧客名                     |
| `phone`          | 文字列   | 必須 | 電話番号                   |
| `phone2`         | 文字列   | 任意 | 補助電話番号               |
| `fax`            | 文字列   | 任意 | FAX番号                    |
| `postal_code`    | 文字列   | 必須 | 郵便番号                   |
| `address`        | 文字列   | 必須 | 住所                       |

### **出力**
#### **正常系**
- **HTTPステータス**: 202 Accepted
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "インポート処理を開始しました。",
    "data": {
        "task_id": "IMPORT_20240301_001",
        "file_name": "customers.xlsx",
        "record_count": 100
    }
}
```

#### **異常系**
- **ファイル形式エラー時**:
  - HTTPステータス: 400 Bad Request
  - レスポンス例:
```json
{
    "status": "error",
    "message": "ファイル形式が不正です。",
    "errors": {
        "file": ["xls, xlsx, csv, txt形式のファイルを指定してください。"]
    }
}
```

## **4. ロジック/処理フロー**
1. ファイルアップロード処理
   - ファイルの形式チェック
   - ファイルサイズのバリデーション
   - 一時ファイルとして保存

2. バックグラウンド処理の開始
   - ファイルの読み込み
   - データの形式チェック
   - 重複チェック
   - バリデーション
   - データの登録
   - 処理結果の記録

3. 処理結果の通知
   - 成功件数、失敗件数の集計
   - エラーログの生成
   - オペレータへの通知

## **5. 関連テーブル/データ構造**
- **`import_tasks` テーブル**
  | カラム名         | 型         | 説明                       |
  |------------------|------------|----------------------------|
  | `id`             | INT        | ユニークID                |
  | `task_id`        | VARCHAR(32)| タスクID                  |
  | `file_name`      | VARCHAR(255)| ファイル名               |
  | `status`         | VARCHAR(16)| 処理状態                  |
  | `total_count`    | INT        | 処理対象レコード数        |
  | `success_count`  | INT        | 成功件数                  |
  | `error_count`    | INT        | 失敗件数                  |
  | `started_at`     | DATETIME   | 処理開始日時              |
  | `completed_at`   | DATETIME   | 処理完了日時              |

## **6. バリデーション仕様**
### ファイルバリデーション
- 拡張子: xls, xlsx, csv, txt
- サイズ: 最大10MB
- 文字コード: UTF-8またはShift-JIS

### データバリデーション
- `name`: 必須、最大64文字
- `phone`: 必須、電話番号形式（数字とハイフンのみ）
- `phone2`: 任意、電話番号形式（数字とハイフンのみ）
- `fax`: 任意、電話番号形式（数字とハイフンのみ）
- `postal_code`: 必須、郵便番号形式（例：123-4567）
- `address`: 必須、最大128文字

## **7. エラーハンドリング**
- ファイル形式が不正な場合、400エラーを返却
- ファイルサイズ超過時、413エラーを返却
- CSRFトークンが無効な場合、419エラーを返却
- サーバーエラー発生時、500エラーを返却
- データ形式エラーは、エラーログに記録して処理を継続

## **8. API仕様**
### **エンドポイント**
- **インポート開始**
  - **URL**: `/operator/customer/import`
  - **HTTPメソッド**: POST

- **処理状況確認**
  - **URL**: `/operator/customer/import/{task_id}/status`
  - **HTTPメソッド**: GET

### **リクエスト例**
```http
POST /operator/customer/import
Content-Type: multipart/form-data

{
    "file": "（ファイルデータ）",
    "_token": "..."
}
```

### **レスポンス例**
#### 処理開始時
```json
{
    "status": "success",
    "message": "インポート処理を開始しました。",
    "data": {
        "task_id": "IMPORT_20240301_001",
        "file_name": "customers.xlsx",
        "record_count": 100
    }
}
```

#### 処理状況確認時
```json
{
    "status": "success",
    "data": {
        "task_id": "IMPORT_20240301_001",
        "progress": 45,
        "total_count": 100,
        "processed_count": 45,
        "success_count": 43,
        "error_count": 2
    }
}
```
