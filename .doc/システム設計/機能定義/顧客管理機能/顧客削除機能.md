# 顧客削除機能設計書

## **1. 機能概要**
- **目的**: オペレータが既存の顧客情報を削除（論理削除）できる機能。
- **対象者**: システムのオペレータ
- **方式**: 論理削除方式（物理削除は行わない）

## **2. 要件一覧**
### **必須要件**
- 顧客情報の論理削除（`deleted_at`の設定）
- 削除前の確認プロセス
- 削除された顧客の認証情報の無効化
- 削除操作のログ記録
- 削除された顧客の復元機能
  - 復元時に認証情報も再有効化
  - 復元操作のログ記録

### **拡張要件**
- 削除理由の記録
- 削除通知のLINEメッセージ送信

## **3. 入出力仕様**
### **入力**
#### 削除時
| 項目             | 型       | 必須 | 説明                       |
|------------------|----------|------|----------------------------|
| `user_code`      | 文字列   | 必須 | 削除対象の顧客コード        |
| `_token`         | 文字列   | 必須 | CSRFトークン               |
| `delete_reason`  | 文字列   | 任意 | 削除理由                   |

#### 復元時
| 項目             | 型       | 必須 | 説明                       |
|------------------|----------|------|----------------------------|
| `user_code`      | 文字列   | 必須 | 復元対象の顧客コード        |
| `_token`         | 文字列   | 必須 | CSRFトークン               |

### **出力**
#### **正常系**
- **HTTPステータス**: 200 OK
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "顧客情報を削除しました。",
    "data": {
        "user_code": "CUST12345",
        "deleted_at": "2024-03-01T10:00:00Z"
    }
}
```

#### **異常系**
- **対象顧客が存在しない場合**:
  - HTTPステータス: 404 Not Found
  - レスポンス例:
```json
{
    "status": "error",
    "message": "指定された顧客が見つかりません。"
}
```

## **4. ロジック/処理フロー**
1. オペレータが削除対象の顧客を選択
2. 削除確認モーダルの表示
3. 削除実行時の処理:
   - CSRFトークンの検証
   - 対象顧客の存在確認
   - 顧客情報の論理削除（`deleted_at`の設定）
   - 関連する認証情報の無効化
   - 操作ログの記録
   - 削除完了モーダルの表示

## **5. 関連テーブル/データ構造**
- **`Users` テーブル**での論理削除
  | カラム名         | 型         | 説明                       |
  |------------------|------------|----------------------------|
  | `deleted_at`     | DATETIME   | 削除日時（NULL=有効）      |

- **`authenticates` テーブル**での認証無効化
  | カラム名         | 型         | 説明                       |
  |------------------|------------|----------------------------|
  | `is_active`      | BOOLEAN    | アカウント有効フラグ       |
  | `deactivated_at` | DATETIME   | 無効化日時                 |

## **6. バリデーション仕様**
- `user_code`: 必須、既存の顧客コード
- `delete_reason`: 任意、最大255文字
- `_token`: 必須、有効なCSRFトークン

## **7. エラーハンドリング**
- 対象顧客が存在しない場合、404エラーを返却
- CSRFトークンが無効な場合、419エラーを返却
- 削除処理に失敗した場合、500エラーを返却
- すでに削除済みの場合、400エラーを返却

## **8. API仕様**
### **エンドポイント**
#### 削除
- **URL**: `/operator/customer/{user_code}`
- **HTTPメソッド**: DELETE

#### 復元
- **URL**: `/operator/customer/{user_code}/restore`
- **HTTPメソッド**: POST

### **リクエスト例**
#### 正常系
##### 削除
```http
DELETE /operator/customer/CUST12345
Content-Type: application/json

{
    "_token": "...",
    "delete_reason": "顧客からの要請により削除"
}
```

##### 復元
```http
POST /operator/customer/CUST12345/restore
Content-Type: application/json

{
    "_token": "..."
}
```

### **レスポンス例**
#### 正常系
##### 削除時
```json
{
    "status": "success",
    "message": "顧客情報を削除しました。",
    "data": {
        "user_code": "CUST12345",
        "deleted_at": "2024-03-01T10:00:00Z"
    }
}
```

##### 復元時
```json
{
    "status": "success",
    "message": "顧客情報を復元しました。",
    "data": {
        "user_code": "CUST12345",
        "restored_at": "2024-03-01T10:00:00Z"
    }
}
```

#### 異常系
##### 削除時
```json
{
    "status": "error",
    "message": "削除処理に失敗しました。",
    "reason": "すでに削除済みの顧客です。"
}
```

##### 復元時
```json
{
    "status": "error",
    "message": "復元処理に失敗しました。",
    "reason": "削除されていない顧客です。"
}
```
