# 顧客登録機能設計書

## **1. 機能概要**
- **目的**: オペレータが新規顧客情報を登録できる機能。
- **対象者**: システムのオペレータ
- **方式**: Webフォームによる手動登録、およびCSVインポートによる一括登録

## **2. 要件一覧**
### **必須要件**
- 顧客の基本情報（氏名、電話番号、住所等）を登録できる
- 登録時に認証用のログインIDとパスワードを自動生成
- 重複登録を防止（電話番号による重複チェック）
- CSVによる一括登録に対応

### **拡張要件**
- 登録完了時にLINEメッセージを送信
- 登録履歴の記録
- 登録時のバリデーションルールのカスタマイズ

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                       |
|------------------|----------|------|----------------------------|
| `name`           | 文字列   | 必須 | 顧客名                     |
| `phone`          | 文字列   | 必須 | 電話番号                   |
| `phone2`         | 文字列   | 任意 | 補助電話番号               |
| `fax`            | 文字列   | 任意 | FAX番号                    |
| `postal_code`    | 文字列   | 必須 | 郵便番号                   |
| `address`        | 文字列   | 必須 | 住所                       |
| `_token`         | 文字列   | 必須 | CSRFトークン               |

### **出力**
#### **正常系**
- **HTTPステータス**: 200 OK
- **レスポンス例**:
```json
{
    "message": "success",
    "login_code": "CUST12345",
    "password": "abcd1234"
}
```

#### **異常系**
- **バリデーションエラー時**:
  - HTTPステータス: 422 Unprocessable Entity
  - レスポンス例:
```json
{
    "message": "error",
    "errors": {
        "phone": ["電話番号の形式が正しくありません。"],
        "postal_code": ["郵便番号は必須です。"]
    }
}
```

## **4. ロジック/処理フロー**
1. オペレータが顧客情報をフォームに入力
2. フロントエンドでの入力値バリデーション
3. バリデーション成功時、確認モーダルを表示
4. 登録実行時の処理:
   - CSRFトークンの検証
   - サーバーサイドでのバリデーション
   - 顧客情報の登録（`Users`テーブル）
   - 認証情報の生成と登録（`authenticates`テーブル）
   - 登録完了モーダルの表示（ログインID・パスワードを含む）

## **5. 関連テーブル/データ構造**
- **`Users` テーブル**
  | カラム名         | 型         | 説明                       |
  |------------------|------------|----------------------------|
  | `id`             | INT        | ユニークID                |
  | `user_code`      | VARCHAR(64)| 顧客コード（ユニーク）     |
  | `site_id`        | INT        | サイトID                  |
  | `name`           | VARCHAR(64)| 顧客名                    |
  | `postal_code`    | VARCHAR(10)| 郵便番号                  |
  | `address`        | VARCHAR(128)| 住所                     |
  | `phone`          | VARCHAR(24)| 電話番号                  |
  | `phone2`         | VARCHAR(24)| 補助電話番号              |
  | `fax`            | VARCHAR(24)| FAX番号                   |
  | `created_at`     | DATETIME   | 作成日時                  |
  | `updated_at`     | DATETIME   | 更新日時                  |
  | `deleted_at`     | DATETIME   | 削除日時                  |

## **6. バリデーション仕様**
- `name`: 必須、最大64文字
- `phone`: 必須、電話番号形式（数字とハイフンのみ）
- `phone2`: 任意、電話番号形式（数字とハイフンのみ）
- `fax`: 任意、電話番号形式（数字とハイフンのみ）
- `postal_code`: 必須、郵便番号形式（例：123-4567）
- `address`: 必須、最大128文字

## **7. エラーハンドリング**
- 入力値が不正な場合、422エラーを返却
- 電話番号が重複している場合、エラーメッセージを表示
- CSRFトークンが無効な場合、419エラーを返却
- サーバーエラー発生時、500エラーを返却

## **8. API仕様**
### **エンドポイント**
- **URL**: `/operator/customer`
- **HTTPメソッド**: POST

### **リクエスト例**
```http
POST /operator/customer
Content-Type: application/json

{
    "name": "山田太郎",
    "phone": "090-1234-5678",
    "phone2": "03-1234-5678",
    "fax": "03-1234-5679",
    "postal_code": "123-4567",
    "address": "東京都渋谷区...",
    "_token": "..."
}
```

### **レスポンス例**
#### 正常系
```json
{
    "message": "success",
    "login_code": "CUST12345",
    "password": "abcd1234"
}
```

#### 異常系
```json
{
    "message": "error",
    "reason": "入力内容に誤りがあります。"
}
```
