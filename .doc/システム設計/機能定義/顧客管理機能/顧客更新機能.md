# 顧客更新機能設計書

## **1. 機能概要**
- **目的**: オペレータが既存の顧客情報を更新できる機能。
- **対象者**: システムのオペレータ
- **方式**: Webフォームによる対話的更新

## **2. 要件一覧**
### **必須要件**
- 顧客の基本情報（氏名、電話番号、住所等）の更新
- 更新前の確認プロセス
- 更新履歴の記録
- 電話番号変更時の重複チェック
- 更新完了時の通知表示

### **拡張要件**
- 更新内容のLINEメッセージ通知
- 一括更新機能（CSVインポート）
- 更新前後の差分表示

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                       |
|------------------|----------|------|----------------------------|
| `user_code`      | 文字列   | 必須 | 更新対象の顧客コード        |
| `name`           | 文字列   | 任意 | 顧客名                     |
| `phone`          | 文字列   | 任意 | 電話番号                   |
| `phone2`         | 文字列   | 任意 | 補助電話番号               |
| `fax`            | 文字列   | 任意 | FAX番号                    |
| `postal_code`    | 文字列   | 任意 | 郵便番号                   |
| `address`        | 文字列   | 任意 | 住所                       |
| `_token`         | 文字列   | 必須 | CSRFトークン               |

### **出力**
#### **正常系**
- **HTTPステータス**: 200 OK
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "顧客情報を更新しました。",
    "data": {
        "user_code": "CUST12345",
        "name": "山田太郎",
        "phone": "090-1234-5678",
        "updated_at": "2024-03-01T10:00:00Z"
    }
}
```

#### **異常系**
- **バリデーションエラー時**:
  - HTTPステータス: 422 Unprocessable Entity
  - レスポンス例:
```json
{
    "status": "error",
    "message": "入力内容に誤りがあります。",
    "errors": {
        "phone": ["電話番号の形式が正しくありません。"],
        "postal_code": ["郵便番号の形式が正しくありません。"]
    }
}
```

## **4. ロジック/処理フロー**
1. オペレータが更新対象の顧客情報を表示
2. フォームに更新内容を入力
3. フロントエンドでの入力値バリデーション
4. 更新確認モーダルの表示
5. 更新実行時の処理:
   - CSRFトークンの検証
   - サーバーサイドでのバリデーション
   - 電話番号重複チェック
   - 顧客情報の更新
   - 更新履歴の記録
   - 更新完了モーダルの表示

## **5. 関連テーブル/データ構造**
- **`Users` テーブル**
  | カラム名         | 型         | 説明                       |
  |------------------|------------|----------------------------|
  | `id`             | INT        | ユニークID                |
  | `user_code`      | VARCHAR(64)| 顧客コード（ユニーク）     |
  | `name`           | VARCHAR(64)| 顧客名                    |
  | `postal_code`    | VARCHAR(10)| 郵便番号                  |
  | `address`        | VARCHAR(128)| 住所                     |
  | `phone`          | VARCHAR(24)| 電話番号                  |
  | `phone2`         | VARCHAR(24)| 補助電話番号              |
  | `fax`            | VARCHAR(24)| FAX番号                   |
  | `updated_at`     | DATETIME   | 更新日時                  |

## **6. バリデーション仕様**
- `user_code`: 必須、既存の顧客コード
- `name`: 任意、最大64文字
- `phone`: 任意、電話番号形式（数字とハイフンのみ）
- `phone2`: 任意、電話番号形式（数字とハイフンのみ）
- `fax`: 任意、電話番号形式（数字とハイフンのみ）
- `postal_code`: 任意、郵便番号形式（例：123-4567）
- `address`: 任意、最大128文字

## **7. エラーハンドリング**
- 入力値が不正な場合、422エラーを返却
- 対象顧客が存在しない場合、404エラーを返却
- 電話番号が重複している場合、エラーメッセージを表示
- CSRFトークンが無効な場合、419エラーを返却
- 更新処理に失敗した場合、500エラーを返却

## **8. API仕様**
### **エンドポイント**
- **URL**: `/operator/customer/{user_code}`
- **HTTPメソッド**: PUT

### **リクエスト例**
```http
PUT /operator/customer/CUST12345
Content-Type: application/json

{
    "name": "山田太郎",
    "phone": "090-1234-5678",
    "phone2": "03-1234-5678",
    "fax": "03-1234-5679",
    "postal_code": "123-4567",
    "address": "東京都渋谷区...",
    "_token": "..."
}
```

### **レスポンス例**
#### 正常系
```json
{
    "status": "success",
    "message": "顧客情報を更新しました。",
    "data": {
        "user_code": "CUST12345",
        "name": "山田太郎",
        "phone": "090-1234-5678",
        "updated_at": "2024-03-01T10:00:00Z"
    }
}
```

#### 異常系
```json
{
    "status": "error",
    "message": "更新処理に失敗しました。",
    "errors": {
        "phone": ["この電話番号は既に登録されています。"]
    }
}
```
