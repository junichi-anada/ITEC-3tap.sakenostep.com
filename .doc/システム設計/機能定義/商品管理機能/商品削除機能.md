# 商品削除機能設計書

## **1. 機能概要**
- **目的**: オペレータが既存の商品を削除できる機能。

## **2. 要件一覧**
### **必須要件**
- 商品コードを指定して商品を削除できる。
- 削除は論理削除で行い、データを物理的に削除しない。
- 商品が存在しない場合、エラーメッセージを返却する。

### **拡張要件**
- 削除理由を記録する。
- 複数の商品を一括で削除できる。

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                                   |
|------------------|----------|------|----------------------------------------|
| `item_code`      | 文字列   | 必須 | 削除対象の商品コード                   |

### **出力**
#### **正常系**
- **HTTPステータス**: 200 OK
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "商品を削除しました。",
    "data": {
        "item_code": "A001"
    }
}
```

#### **異常系**
- **対象の商品が存在しない場合**:
  - HTTPステータス: 404 Not Found
  - レスポンス例:
```json
{
    "status": "error",
    "message": "対象の商品が見つかりません。"
}
```
- **入力パラメータが無効な場合**:
  - HTTPステータス: 400 Bad Request
  - レスポンス例:
```json
{
    "status": "error",
    "message": "入力パラメータが不正です。"
}
```

## **4. ロジック/処理フロー**
1. オペレータが削除対象の商品コードをリクエストとして送信。
2. `items` テーブルを検索し、対象の商品が存在するか確認。
    - 存在しない場合、404エラーを返却。
3. 該当商品の `deleted_at` カラムに削除日時を記録。
4. 削除成功時に、商品コードを含むレスポンスを返却。

## **5. 関連テーブル/データ構造**
- **`items` テーブル**
  | カラム名         | 型       | 説明                       |
  |------------------|----------|----------------------------|
  | `id`             | INT      | ユニークID                |
  | `item_code`      | VARCHAR  | 商品コード（ユニーク）     |
  | `name`           | VARCHAR  | 商品名                    |
  | `price`          | DECIMAL  | 商品価格                  |
  | `category_id`    | INT      | 商品カテゴリID            |
  | `published_at`   | DATETIME | 公開日時                  |
  | `is_recommended` | BOOLEAN  | おすすめ商品フラグ        |
  | `created_at`     | DATETIME | 作成日時                  |
  | `updated_at`     | DATETIME | 更新日時                  |
  | `deleted_at`     | DATETIME | 削除日時                  |

## **6. バリデーション仕様**
- `item_code`: 必須、既存の商品コード。

## **7. エラーハンドリング**
- 対象の商品が存在しない場合、404エラーを返却。
- 無効な入力パラメータの場合、400エラーを返却。

## **8. API仕様**
### **エンドポイント**
- **URL**: `/api/items/{item_code}`
- **HTTPメソッド**: DELETE

### **リクエスト例**
```http
DELETE /api/items/A001
```

### **レスポンス例**
#### 正常系
```json
{
    "status": "success",
    "message": "商品を削除しました。",
    "data": {
        "item_code": "A001"
    }
}
```

#### 異常系
```json
{
    "status": "error",
    "message": "対象の商品が見つかりません。"
}
