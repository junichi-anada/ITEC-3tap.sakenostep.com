# 商品データインポート機能設計書

## **1. 機能概要**
- **目的**: オペレータが商品データを一括でインポートできる機能
- **対象者**: システムのオペレータ
- **方式**: ファイルアップロード方式（Excel/CSV/TXT対応）

## **2. 要件一覧**
### **必須要件**
- Excel（xls/xlsx）、テキストファイル（csv/txt）形式のファイルをインポート可能
- ファイルサイズは最大10MBまで
- インポート結果をログとして記録
- インポート処理はバックグラウンドで実行
- 処理結果をオペレータに通知

### **拡張要件**
- インポート履歴の表示
- エラーデータの出力機能
- インポートテンプレートのダウンロード機能

## **3. 入出力仕様**
### **入力**
| 項目       | 型     | 必須 | 説明                                |
|------------|--------|------|-------------------------------------|
| `file`     | File   | 必須 | インポートファイル（10MB以内）      |
| `_token`   | string | 必須 | CSRFトークン                        |

### **出力**
#### **正常系**
- **HTTPステータス**: 202 Accepted
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "インポート処理を開始しました。",
    "data": {
        "task_code": "IMPORT_20240301_001",
        "file_name": "items.xlsx",
        "record_count": 100
    }
}
```

#### **異常系**
- **ファイル形式エラー**:
  - HTTPステータス: 400 Bad Request
```json
{
    "status": "error",
    "message": "ファイル形式が不正です。",
    "errors": {
        "file": ["xls, xlsx, csv, txt形式のファイルを指定してください。"]
    }
}
```

## **4. 処理フロー**
1. ファイルアップロード処理
   - ファイルの形式チェック
   - ファイルサイズのバリデーション
   - 一時ファイルとして保存

2. インポートタスク作成
   - タスクコードの生成
   - インポートタスクの登録
   - ファイルパスの保存

3. バックグラウンド処理
   - ファイルの読み込み
   - データの形式変換
   - バリデーション
   - 商品データの一括登録
   - ログの記録

4. 結果通知
   - 処理完了時にオペレータへ通知
   - エラーがある場合はエラー内容を通知

## **5. データ構造**
### **import_tasks テーブル**
| カラム名         | 型       | 説明                       |
|------------------|----------|----------------------------|
| `id`             | INT      | タスクのユニークID         |
| `task_code`      | VARCHAR  | タスクコード（ユニーク）    |
| `site_id`        | INT      | サイトID                  |
| `data_type`      | VARCHAR  | データ種類（ITEM固定）     |
| `file_path`      | VARCHAR  | ファイルパス              |
| `status`         | ENUM     | 処理状態                  |
| `status_message` | TEXT     | ステータスメッセージ       |
| `operator_id`    | INT      | 実行者ID                  |
| `uploaded_at`    | DATETIME | アップロード日時          |
| `imported_at`    | DATETIME | インポート完了日時        |

### **items テーブル**
| カラム名         | 型       | 説明                       |
|------------------|----------|----------------------------|
| `item_code`      | VARCHAR  | 商品コード（ユニーク）     |
| `name`           | VARCHAR  | 商品名                    |
| `price`          | DECIMAL  | 価格                      |
| `category_id`    | INT      | カテゴリID                |
| `published_at`   | DATETIME | 公開日時                  |
| `from_source`    | ENUM     | 登録方法（IMPORT）         |
| `is_recommended` | BOOLEAN  | おすすめ商品フラグ        |

## **6. バリデーション仕様**
### ファイルバリデーション
- 拡張子: xls, xlsx, csv, txt
- サイズ: 最大10MB
- 文字コード: UTF-8（CSVの場合）

### データバリデーション
- `item_code`: 必須、最大64文字、ユニーク
- `name`: 必須、最大64文字
- `price`: 必須、0以上の数値
- `category_id`: 必須、存在するカテゴリID

## **7. エラーハンドリング**
### ファイルエラー
- 不正な形式: 400 Bad Request
- サイズ超過: 400 Bad Request
- 文字コード不正: 400 Bad Request

### データエラー
- バリデーションエラー: エラーログに記録
- 重複エラー: スキップしてログに記録
- DB登録エラー: トランザクションロールバック

## **8. セキュリティ対策**
- CSRFトークンによる保護
- ファイル拡張子の厳密なチェック
- アップロードサイズの制限
- 実行権限の確認
- 一時ファイルの適切な削除

## **9. パフォーマンス対策**
- バックグラウンドジョブの使用
- バッチ処理（1000件単位）
- インデックスの活用
- 一括INSERT処理の採用

## **10. テストケース**
### 正常系
1. 正常なExcelファイルのインポート
2. 正常なCSVファイルのインポート
3. 大量データ（1000件以上）のインポート

### 異常系
1. 不正な形式のファイル
2. サイズ超過のファイル
3. 必須項目が欠落したデータ
4. 重複データを含むファイル
