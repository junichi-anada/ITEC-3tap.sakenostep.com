# 商品更新機能設計書

## **1. 機能概要**
- **目的**: オペレータが既存の商品情報を更新できる機能。

## **2. 要件一覧**
### **必須要件**
- 商品名、価格、カテゴリ、公開日時、おすすめ商品の可否を更新できる。
- 商品コードは変更不可。
- 商品が存在しない場合、エラーメッセージを返却する。

### **拡張要件**
- 更新時に商品画像を変更できる。
- 更新履歴を記録する。

## **3. 入出力仕様**
### **入力**
| 項目             | 型       | 必須 | 説明                                   |
|------------------|----------|------|----------------------------------------|
| `item_code`      | 文字列   | 必須 | 更新対象の商品コード                   |
| `name`           | 文字列   | 任意 | 商品名（変更する場合のみ指定）         |
| `price`          | 小数     | 任意 | 商品価格（変更する場合のみ指定）       |
| `category_id`    | 整数     | 任意 | 商品カテゴリID（変更する場合のみ指定） |
| `published_at`   | 日付     | 任意 | 公開開始日時（変更する場合のみ指定）   |
| `is_recommended` | ブール型 | 任意 | おすすめ商品かどうか                   |

### **出力**
#### **正常系**
- **HTTPステータス**: 200 OK
- **レスポンス例**:
```json
{
    "status": "success",
    "message": "商品情報を更新しました。",
    "data": {
        "item_code": "A001",
        "name": "商品A",
        "price": 1200,
        "category_id": 2,
        "published_at": "2025-01-02",
        "is_recommended": true
    }
}
```

#### **異常系**
- **対象の商品が存在しない場合**:
  - HTTPステータス: 404 Not Found
  - レスポンス例:
```json
{
    "status": "error",
    "message": "対象の商品が見つかりません。"
}
```
- **入力パラメータが無効な場合**:
  - HTTPステータス: 400 Bad Request
  - レスポンス例:
```json
{
    "status": "error",
    "message": "入力パラメータが不正です。"
}
```

## **4. ロジック/処理フロー**
1. オペレータが更新対象の商品コードと変更内容をリクエストとして送信。
2. `items` テーブルを検索し、対象の商品が存在するか確認。
    - 存在しない場合、404エラーを返却。
3. 入力データのバリデーションを実施。
    - 無効なデータの場合、400エラーを返却。
4. バリデーションに成功した場合、指定されたカラムのみ更新。
5. 更新後のデータをレスポンスとして返却。

## **5. 関連テーブル/データ構造**
- **`items` テーブル**
  | カラム名         | 型       | 説明                       |
  |------------------|----------|----------------------------|
  | `id`             | INT      | ユニークID                |
  | `item_code`      | VARCHAR  | 商品コード（ユニーク）     |
  | `name`           | VARCHAR  | 商品名                    |
  | `price`          | DECIMAL  | 商品価格                  |
  | `category_id`    | INT      | 商品カテゴリID            |
  | `published_at`   | DATETIME | 公開日時                  |
  | `is_recommended` | BOOLEAN  | おすすめ商品フラグ        |
  | `created_at`     | DATETIME | 作成日時                  |
  | `updated_at`     | DATETIME | 更新日時                  |

## **6. バリデーション仕様**
- `item_code`: 必須、既存の商品コード。
- `name`: 最大100文字。
- `price`: 0以上の値。
- `category_id`: 正の整数。
- `published_at`: 有効な日付形式。
- `is_recommended`: 真偽値。

## **7. エラーハンドリング**
- 対象の商品が存在しない場合、404エラーを返却。
- 無効な入力パラメータの場合、400エラーを返却。

## **8. API仕様**
### **エンドポイント**
- **URL**: `/api/items/{item_code}`
- **HTTPメソッド**: PUT

### **リクエスト例**
```json
{
    "name": "商品A",
    "price": 1200,
    "category_id": 2,
    "published_at": "2025-01-02",
    "is_recommended": true
}
```

### **レスポンス例**
#### 正常系
```json
{
    "status": "success",
    "message": "商品情報を更新しました。",
    "data": {
        "item_code": "A001",
        "name": "商品A",
        "price": 1200,
        "category_id": 2,
        "published_at": "2025-01-02",
        "is_recommended": true
    }
```
