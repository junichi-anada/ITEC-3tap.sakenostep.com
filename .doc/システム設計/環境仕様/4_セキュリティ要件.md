# 4. セキュリティ要件

## 4.1 認証・認可

### トークン認証
- JWT（JSON Web Token）の使用
  - トークン有効期限: 24時間
  - リフレッシュトークン: 30日
  - 署名アルゴリズム: HS256

### ロールベースアクセス制御
1. 管理者ロール
   - 全機能へのアクセス権限
   - ユーザー管理権限
   - システム設定権限

2. オペレータロール
   - 商品管理権限
   - 注文管理権限
   - 顧客対応権限

3. 閲覧専用ロール
   - 各種情報の閲覧のみ
   - 更新・削除権限なし

### セッション管理
- セッションタイムアウト: 30分
- 同時ログイン制限: 3セッションまで
- セッションハイジャック対策
  - セッションIDの再生成
  - HTTPSの強制

## 4.2 データ保護

### 暗号化
- パスワード: bcryptによるハッシュ化
- API通信: TLS 1.3
- 機密データ: AES-256

### 入力値検証
- XSS対策
  - HTMLエスケープ処理
  - CSP（Content Security Policy）の実装
- SQLインジェクション対策
  - プリペアドステートメントの使用
  - パラメータバインディング

### アクセス制御
- IPアドレス制限
  - 管理画面へのアクセス制限
  - API制限
- レート制限
  - API呼び出し: 60回/分
  - ログイン試行: 5回/30分

## 4.3 監査ログ

### セキュリティログ
- ログイン/ログアウト
- 権限変更
- データアクセス
- エラー発生

### 保管期間
- セキュリティログ: 1年
- アクセスログ: 3ヶ月
- エラーログ: 6ヶ月

### ログフォーマット
```json
{
  "timestamp": "ISO8601形式",
  "event_type": "イベント種別",
  "user_id": "操作ユーザーID",
  "ip_address": "IPアドレス",
  "action": "実行アクション",
  "status": "結果ステータス",
  "details": "詳細情報"
}
```
