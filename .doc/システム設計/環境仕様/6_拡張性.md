## 6.2 インターフェース

### REST API
- エンドポイント命名規則
  ```
  /api/v1/{resource}/{action}
  例: /api/v1/items/search
  ```
- レスポンスフォーマット
  ```json
  {
    "status": "success|error",
    "message": "処理メッセージ",
    "data": {
      // レスポンスデータ
    },
    "meta": {
      "total": 100,
      "current_page": 1
    }
  }
  ```
- エラーハンドリング
  - HTTPステータスコード
  - エラーメッセージの統一
