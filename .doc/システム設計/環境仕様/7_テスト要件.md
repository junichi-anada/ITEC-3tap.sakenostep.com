# 7. テスト要件

## 7.1 テストレベル

### 単体テスト
- 対象
  - サービスクラス
  - モデルクラス
  - ユーティリティクラス
- ツール
  - PHPUnit
  - PHPStan（静的解析）
- カバレッジ目標: 80%以上

### 統合テスト
- API結合テスト
  - エンドポイントの動作確認
  - リクエスト/レスポンスの検証
  - エラーハンドリングの確認
- データベース結合テスト
  - トランザクション処理
  - データの整合性確認

### E2Eテスト
- ツール: Laravel Dusk
- テスト項目
  - ユーザー認証フロー
  - 商品管理フロー
  - 注文処理フロー
- ブラウザ対応
  - Chrome
  - Firefox
  - Safari

## 7.2 テスト環境

### 開発環境
- ローカル開発環境
  - Docker環境
  - テストデータセット

### ステージング環境
- 本番相当環境
  - 本番同等の設定
  - マスクされた本番データ

### CI/CD環境
- GitHub Actions
  - プルリクエスト時の自動テスト
  - マージ時の自動デプロイ

## 7.3 テストデータ

### マスターデータ
- 商品データ
- カテゴリデータ
- ユーザーデータ

### テストケース
```yaml
# テストケース例
case:
  name: "商品登録_正常系"
  input:
    item_code: "ITEM001"
    name: "テスト商品"
    price: 1000
  expected:
    status: 200
    message: "商品を登録しました"
```
