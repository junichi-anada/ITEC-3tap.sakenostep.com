# 5. 運用要件

## 5.1 バックアップ

### データベースバックアップ
1. フルバックアップ
   - 実行頻度: 毎日
   - 実行時間: 深夜1時
   - 保持期間: 30日
   - 保存場所: AWS S3

2. 差分バックアップ
   - 実行頻度: 6時間ごと
   - 保持期間: 7日
   - 保存場所: AWS S3

### ログファイル
- 保管期間
  - アプリケーションログ: 3ヶ月
  - アクセスログ: 3ヶ月
  - エラーログ: 6ヶ月
- ローテーション
  - 日次でローテーション
  - 月次でアーカイブ

## 5.2 監視

### システム監視
- 監視項目
  - CPU使用率: 80%以上でアラート
  - メモリ使用率: 85%以上でアラート
  - ディスク使用率: 90%以上でアラート
- 監視間隔
  - リソース監視: 5分間隔
  - プロセス監視: 1分間隔

### アプリケーション監視
- エラー監視
  - 5xx エラー発生時にアラート
  - 認証失敗の急増検知
- パフォーマンス監視
  - レスポンスタイム: 3秒以上で警告
  - スロークエリ: 1秒以上で記録

### 通知設定
- 通知方法
  - Slack
  - メール
  - LINE Notify
- 重要度レベル
  - 緊急: 即時対応必要
  - 警告: 24時間以内に対応
  - 情報: 定期確認

## 5.3 保守

### 定期メンテナンス
- 実施頻度: 月1回
- 実施時間: 深夜1時〜3時
- 作業内容
  - セキュリティパッチ適用
  - ミドルウェアアップデート
  - パフォーマンスチューニング

### 障害対応
- 対応フロー
  1. 障害検知
  2. 一次切り分け
  3. エスカレーション判断
  4. 対応実施
  5. 報告書作成
- 優先度定義
  - P1: サービス停止（即時対応）
  - P2: 重要機能障害（3時間以内）
  - P3: 軽微な障害（24時間以内）
