# 2.1 フレームワーク仕様

## 基本フレームワーク
- Laravel 11.0+
- PHP 8.3+

## ミドルウェア構成
| 名称 | パッケージ名 | 用途 |
|------|--------------|------|
| 認証 | sanctum | APIトークン認証 |
| CORS | fruitcake/laravel-cors | クロスオリジン制御 |
| キャッシュ | redis | データキャッシュ |

## 主要パッケージ
| パッケージ名 | バージョン | 用途 |
|--------------|------------|------|
| laravel/breeze | ^1.0 | 認証基盤 |
| spatie/laravel-permission | ^5.0 | 権限管理 |

## 採用する設計パターン
1. Repository Pattern
   - データアクセス層の抽象化
   - テスタビリティの向上

2. Service Layer Pattern
   - ビジネスロジックの分離
   - 再利用性の向上

3. Factory Pattern
   - オブジェクト生成の一元管理
   - 依存性の制御
