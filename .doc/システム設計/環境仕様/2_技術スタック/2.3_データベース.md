# 2.3 データベース仕様

## データベースエンジン
- MySQL 8.0+
  - 文字コード: UTF-8
  - 照合順序: utf8mb4_unicode_ci

## インデックス戦略
1. 主要検索キー
   - 商品コード
   - 顧客コード
   - 注文番号
   
2. 複合インデックス
   - 商品検索: (category_id, published_at)
   - 注文検索: (user_id, ordered_at)

3. 全文検索インデックス
   - 商品名
   - 商品説明

## パーティション戦略
1. 注文テーブル
   - RANGE パーティション
   - パーティションキー: ordered_at
   - 保持期間: 3年

2. ログテーブル
   - RANGE パーティション
   - パーティションキー: created_at
   - 保持期間: 1年

## バックアップ戦略
1. フルバックアップ
   - 実行頻度: 毎日
   - 実行時間: 深夜1時
   - 保持期間: 30日

2. 差分バックアップ
   - 実行頻度: 6時間ごと
   - 保持期間: 7日

## パフォーマンスチューニング
1. クエリ最適化
   - インデックスの適切な使用
   - JOINの最適化
   - サブクエリの最小化

2. コネクション管理
   - 最大接続数: 1000
   - タイムアウト: 60秒
